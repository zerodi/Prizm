INNER & OUTER INPUT

C целью поддержания 2 паттернов выработанного пользовательского опыта в существующих системах все формы ввода имеют 2 вида:
Формы ввода, у которых подпись (label) находится внутри формы (INNER INPUT)
Формы ввода, у которых подпись (label) находится вне формы (OUTER INPUT)

Новые системы будут создаваться с применением 1 паттерна, что со временем избавит нас от “дублей” компонентов. Используйте 1 тип форм ввода в рамках приложения / семейства приложений.

Sizes

В нашей ДС используется размерность форм ввода: L, M, S.
В зависимости от типа (inner / outer), высота формы составляет:

L M S
INNER 44px 36px
OUTER 40px 32px 24px

Status

В нашей ДС предусмотрены 4 варианта статуса формы ввода:
Default - обычное состояние компонента (описано выше)
Success
Warning
Danger
Смена статуса формы ввода происходит согласно принципам валидации форм.

Success

Статус Success возможен только для заполненных форм и означает корректность введенных значений.
Используйте этот статус только в тех ситуациях, когда пользователю необходимо дать обратную связь об успешном вводе данных. В остальных случях - используйте статус Default, чтобы избежать лишего информационного и визуального шума.

Warning

Статус Warning - предупреждение о том, что значение формы (введенное значение или “пусто”) может повлиять на отклик / работу / расчет системы. При этом, корректировка значений пользователем - не обязательна.

Danger

Статус Danger - сообщение пользователю о том, что значение формы (введенное значение или “пусто”): критично для исполнения сценария (если поле не редактируемо / только для чтения) сценарий выполнить невозможно (обязательны ввод или корректировка введенных значений).

Validation

Сообщение об ошибке
1. Иконка.
При наведении на иконку статуса появляется всплывающее сообщение (hint) об ошибке / рекомендациях по исправлению. Используйте этот способ для оптимизации полезного пространства рабочей области. 2. Подсказка снизу.
Такой способ наиболее эффективен в плане скорости сообщения пользователю, но “съедает” пространство, которое мы закладываем под подсказку. !!! Место под подсказку должно быть заложено на этапе проектирования - не допустимо смещение контента под вывод сообщения.
